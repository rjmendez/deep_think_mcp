KEY FEATURES
═══════════════════════════════════════════════════════════════════════════════

✓ Batching
  - Size-based: Publishes when N findings queued (configurable, default 10)
  - Time-based: Publishes after T milliseconds (configurable, default 5000ms)
  - Per-device: Separate batches per device_id

✓ Retry Strategy
  - Exponential backoff: 1s, 2s, 4s, 8s wait times
  - Max retries: Configurable (default 8)
  - Then: Persist to SQLite for later recovery

✓ Persistence
  - SQLite database: ~/.deep_think/findings_queue.db
  - Auto-init on first use
  - Survives application restarts
  - Confirmations also persisted

✓ Auto-Recovery
  - Load persisted findings on startup
  - Replay with small delays between batches
  - Delete successfully published findings

✓ MQTT Publishing
  - Topic: dama/colony/findings/{device_id}
  - QoS: 1 (at least once)
  - Payload: JSON array of Finding objects

✓ Subscription
  - Topic: dama/{device_id}/anomaly_confirmation
  - Status: confirmed | rejected | uncertain
  - Callback pattern for confirmations

✓ Type System
  - 100% type hints on all functions and classes
  - Async/await throughout
  - Comprehensive docstrings

✓ Error Handling
  - Graceful degradation (continues without MQTT)
  - Comprehensive logging (debug, info, warning, error)
  - No exceptions leak to caller

USAGE QUICK START
═══════════════════════════════════════════════════════════════════════════════

1. Initialize Publisher
   from mqtt_findings_publisher import MQTTFindingsPublisher, load_config_from_env
   
   config = load_config_from_env()
   publisher = MQTTFindingsPublisher(**config)
   await publisher.start()

2. Extract Findings
   from mqtt_findings_publisher import findings_from_deep_think_result
   
   findings = findings_from_deep_think_result(
       deep_think_result,
       device_id="ant_001",
       anomaly_threshold=0.5
   )

3. Publish Findings
   for finding in findings:
       await publisher.publish_finding(finding)

4. Shutdown
   await publisher.stop()  # Flushes pending batches
