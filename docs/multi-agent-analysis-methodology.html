Multi-Agent AI Analysis
Hallucination Mitigation Framework

A structural approach to trustworthy AI-generated analysis
The core idea in one sentence: When a single AI model generates a conclusion and evaluates it, errors go unchallenged. This framework separates those roles — using multiple AI agents with opposing mandates to cross-examine each other's reasoning — so errors are structurally caught rather than accidentally avoided.

Reference implementation: deep-think-mcp is an MCP server that applies this methodology as a multi-pass reasoning engine with task-class routing, provider selection, and async job execution. See Section 10 for the implementation reference.

1 The Problem: AI Models That Grade Their Own Work

When you ask a single AI model to analyze a document, the same process that generates the answer also decides whether the answer is correct. This is like asking an author to be their own editor — on a deadline, with no external review. The model's confidence and the model's accuracy are not the same thing.

Figure 1 — The single-agent feedback loop: the model that generates the answer also validates it.

What this looks like in practice

Confident Confabulation
States something plausible but factually wrong with full confidence — no mechanism to catch it.
Source Bleed
Attributes a finding to Source A when it actually came from Source B — subtle, hard to spot in review.
Inference Drift
Presents a reasonable interpretation as a confirmed conclusion — the gap between "suggests" and "confirms."

2 The Solution: Structural Separation of Roles

The fix is architectural. Instead of asking one model to both generate and validate, we separate those functions across agents with different — and deliberately opposing — mandates. No agent evaluates its own work. Every conclusion must survive a challenge from a peer agent designed to find its weaknesses.

Figure 2 — Two-phase architecture: diverging analysis followed by orthogonal verification. Both phases run in parallel within their own fan-out.

3 Pattern 1 — Perspective Fan-Out

The same source data is given to multiple agents simultaneously. Each agent has a structurally different mandate — one designed to find exonerating evidence, another to build the strongest case against, another focused purely on evidence quality, another on organizational risk. Because their goals oppose each other, each agent's blind spots are another agent's primary focus.

Figure 3 — Each agent receives identical source data but a different analytical mandate. Opposing goals surface blind spots that a single agent would miss.

Why opposing mandates work better than "be thorough"

A defense agent is required by its mandate to find the innocent explanation — it cannot rationalize away a weak signal the way a "balanced" agent can. A prosecution agent must assume the worst-case reading. Because they are forced into opposite corners, the gaps in one agent's argument become the other's main finding. The synthesis step doesn't reconcile them — it uses their disagreement as a signal.

4 Convergence = Confidence

When agents with opposing mandates independently reach the same conclusion, that convergence is meaningful. It means the finding survived a challenge designed to defeat it. Items where all agents agree are your highest-confidence action items. Divergence tells you where the evidence is genuinely ambiguous.

Figure 4 — The overlap of four opposing agents is a trust signal. Where all four independently flag the same item, that finding is structurally robust.

5 Pattern 2 — Fact-Check Fan-Out

After Phase 1 produces a draft document, a second fan-out runs three reviewers against it simultaneously. Each reviewer gets the draft and the original source data. The source data is the ground truth. The reviewer's job is to find conflicts between the document and that ground truth — not to use their own knowledge or judgment about what "seems right."

Figure 5 — The fact-check fan-out. Three reviewers with orthogonal mandates each receive the draft document plus the original source data as ground truth.

6 Pattern 3 — Evidence Quality Labeling

Before any analysis begins, every piece of input data is labeled with its confidence level. This forces the AI — and the humans reading the output — to distinguish between what is confirmed and what is assumed. It also gives the attribution reviewer a clear basis for flagging when a low-confidence claim is being cited as if it were verified.

✅ VERIFIED
Directly confirmed by a tool query, database lookup, or independent system call during this analysis session. The highest trust level.
📋 PROVIDED
Supplied in raw data by a human analyst or external report. Taken at face value but not re-queried or independently cross-checked.
⚠ INFERRED
An analyst interpretation or logical deduction from available data. May be reasonable but is not confirmed. Treat as hypothesis.
❌ UNKNOWN
No data available. This is a gap — the question cannot be answered from current sources. Must be explicitly surfaced, not silently omitted.

Figure 6 — The four confidence tiers. Labels appear inline in the document so readers and reviewers can instantly assess trust without tracing back through the analysis.

7 deep_think_mcp — Multi-Pass Reasoning Engine

deep_think_mcp implements this framework as an MCP server with an async job queue. Each call runs a question through four structurally different reasoning passes in sequence — each re-encoding the problem from a different analytical surface — before synthesis commits to a final answer. Task-class routing selects specialized pass directives and model recommendations based on query type.

Figure 7 — deep_think_mcp's four sequential passes (general task class). Pass 3 is the critical adversarial pass — argues against the conclusion before synthesis commits. Pass directives and specialist models change with task_class; see Section 10.

Async Job Pattern

Passes run asynchronously — submit a job and poll for completion. Cloud providers typically return in 30–90 s; local Ollama in 2–5 min.

# 1. Submit
deep_think_async(question="…", task_class="investigation", passes=3)
→ { job_id: "abc-123" }

# 2. Poll (or wait for completion notification)
get_thinking_result(job_id="abc-123")
→ { status: "complete", final_answer: "…", reasoning_chain: [{pass: 1, …}, …] }

8 Which Pattern Catches Which Hallucination

The three hallucination types are not equally likely in all situations, and they are caught by different parts of the framework. Understanding this mapping helps you choose the right pattern for your use case.

Hallucination Type What it looks like Caught by
Confident Confabulation States a specific fact (number, date, name) that is plausible but wrong. The model's confidence is high. Nothing in the output signals the error. Fact Checker — compares every specific claim against source data directly. Confabulated facts conflict with ground truth.
Source Bleed Attributes a finding to Source A when it actually came from Source B. Common when multiple sources are in the same context window. Attribution Reviewer — explicitly checks which source each claim derives from and flags misattribution or unqualified single-source claims.
Inference Drift Presents a plausible interpretation as a confirmed conclusion. The gap between "this suggests" and "this confirms" disappears. Adversarial Agent + Attribution Reviewer — the adversarial agent's mandate is to find over-reaches; the attribution reviewer flags inferences stated as conclusions.
Omission Silently skips a gap rather than flagging it. "No evidence found" where data was never actually queried. Forensics / Compliance Agents — their mandates require surfacing what hasn't been checked, not just what has. Gaps are findings.

Key insight: No single pass catches all four types. This is why structural separation matters — each failure mode requires a different adversarial posture to surface it. A single "review for accuracy" prompt leaves two or three failure modes unaddressed.

9 Implementation Principles

These principles apply regardless of which AI platform or model provider is used. They describe the structural requirements for the approach to work.

  • Agents must be stateless and isolated. Each agent receives the full context in its prompt. Agents share no state with each other. Shared state allows one agent's errors to contaminate another's analysis — exactly what this pattern prevents.
  • Ground truth must be explicit in every prompt. Providing source data inside the agent's prompt (not just the document to review) transforms fact-checking from recall to comparison. Comparison is structurally much harder to hallucinate on.
  • Mandate clarity beats mandate breadth. A focused mandate ("find every overreach") outperforms a broad one ("review for accuracy"). Narrow mandates reduce the surface area where an agent can rationalize away a finding it's uncomfortable making.
  • Run fan-out agents in parallel. Agents with independent mandates have no dependencies on each other. Parallel execution keeps total latency close to the time of a single agent — not N × single agent.
  • Escalate model capability at synthesis. Earlier passes can use faster, cheaper models. The synthesis pass — where all prior reasoning must be integrated and contradictions resolved — benefits most from the highest-capability model available.
  • Divergence is signal, not failure. When agents disagree, that disagreement identifies exactly where the evidence is genuinely ambiguous. Do not resolve divergence by averaging — present it to the human decision-maker with the agents' reasoning intact.
  • Label the taxonomy before analysis begins. Applying Verified / Provided / Inferred / Unknown labels to input data before spawning agents gives every downstream agent and human reader a consistent trust baseline. Unlabeled data will be cited at the wrong confidence level.

10 deep-think-mcp — Task Class & API Reference

Task classes select specialized pass directives and model recommendations. Specify task_class when calling deep_think_async. Default is general. All classes run 4 passes unless overridden with the passes parameter (2–6).
